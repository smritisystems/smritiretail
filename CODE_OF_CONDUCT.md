# Code of Conduct

## Our Commitment

We are committed to fostering an open, inclusive, respectful, and collaborative community where everyone can contribute, learn, and grow regardless of experience level, background, or identity.

We believe that constructive collaboration and professional conduct are essential to building reliable enterprise software.

---

# Our Standards

Examples of behavior that contributes to a positive environment include:

- Being respectful and professional.
- Welcoming new contributors.
- Providing constructive feedback.
- Accepting feedback gracefully.
- Helping others learn.
- Respecting different viewpoints and experiences.
- Focusing discussions on technical and business value.
- Showing empathy and patience during reviews.

---

# Unacceptable Behavior

The following behaviors are unacceptable:

- Harassment or discrimination of any kind.
- Personal attacks or insults.
- Hate speech or abusive language.
- Threats or intimidation.
- Trolling or deliberately disruptive behavior.
- Publishing private information without permission.
- Spam or promotional content unrelated to the project.
- Malicious code submissions.
- Knowingly submitting insecure or harmful code.
- Any behavior that creates an unsafe or hostile environment.

---

# Professional Discussions

Disagreements are natural in software engineering.

When disagreements arise:

- Discuss ideas—not individuals.
- Support proposals with evidence.
- Remain respectful.
- Accept maintainers' final decisions.

The project's success takes priority over personal preferences.

---

# Code Review Expectations

When reviewing code:

- Review the code—not the contributor.
- Explain requested changes clearly.
- Be constructive and objective.
- Recognize good work.
- Keep discussions technical and respectful.

---

# Community Responsibilities

Contributors are expected to:

- Follow project documentation.
- Respect project architecture.
- Write maintainable code.
- Keep pull requests focused.
- Report bugs responsibly.
- Protect user privacy and security.
- **Not bypass the SMRITI Platform Abstraction Layer (PAL).** All platform and data access must route through the SMRITI Core Framework (`from smriti_retail_os import smriti`). Direct low-level adapter calls are permitted only within `core/platform/`. See `ARCHITECTURE.md §15` for the canonical patterns.

---

# Security

Please do not publicly disclose security vulnerabilities.

Instead:

- Contact the maintainers privately.
- Provide reproduction steps.
- Allow reasonable time for investigation and fixes.

Responsible disclosure helps protect everyone.

---

# Enforcement

Project maintainers are responsible for enforcing this Code of Conduct.

Maintainers may:

- Request changes.
- Remove inappropriate content.
- Close discussions.
- Reject pull requests.
- Suspend or ban contributors who repeatedly violate these standards.

Enforcement decisions will be fair, consistent, and made in the best interest of the project and community.

---

# Scope

This Code of Conduct applies to:

- GitHub Issues
- Pull Requests
- Discussions
- Documentation
- Community spaces
- Project communication channels

---

# Reporting Issues

If you experience or witness behavior that violates this Code of Conduct, please report it privately to the project maintainers.

Include:

- A description of the incident
- Relevant screenshots or links (if available)
- Any additional context that may help the investigation

Reports will be handled confidentially whenever possible.
